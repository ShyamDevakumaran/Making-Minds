Work Process

  • START
  • BUILD
  • REVIEW
  • TESTING
  • DEPLOY

START

Create / Start the project with the Project Plan , Resource Plan , Requirements , Technologies to be used etc.. And ready to start the project.

BUILD

In Bulid phase bring up our team together and starts to developing the project with the gathered Resources and calculates the estimated time to complete the build phase.

REVIEW

A standard review process will consist of three phases: Initiation, Research, and Report. Two standard reviews are planned for each project. A special review maybe held if the project is non-compliant or at the request of a Stakeholder.

TESTING

The primary purpose of the Testing Phase is to determine whether the automated system/application software or other IT solution developed or acquired and preliminarily tested during the Testing Phase is ready for Deployment.

DEPLOY

The deployment phase is the final phase of the SDLC and puts the product into production. After the Testing team tests the product and the product passes each testing phase, the product is ready to go live.
